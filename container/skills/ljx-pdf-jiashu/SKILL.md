# 家书 PDF 处理技能

## 功能概述

当检测到文件名包含"家书"的PDF文件时，使用本技能将PDF内容按标准模板进行总结，并保存到知识库的对应目录。

## 处理流程

### Step 1: 读取 PDF 内容

**必须使用 PyMuPDF (fitz) 直接提取文本，禁止使用 Read 工具读取 PDF。**

> ⚠️ **警告**：Read 工具可能存在内容匹配/缓存机制，读取的内容可能与PDF实际内容不符。务必使用 fitz 直接读取才能保证内容正确。

使用 fitz 提取文本，**必须强制 UTF-8 输出**：

```python
import fitz, io, sys
sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
doc = fitz.open('家书文件.pdf')
for i, page in enumerate(doc):
    print(f'=== 第{i+1}页 ===')
    print(page.get_text())
```

> **注意**：乱码是 Windows 控制台 GBK 显示问题，不是真正的编码错误。PDF 文本实际是正确的 UTF-8 中文。强制 stdout UTF-8 后即可正常显示。

**重要**：读取后必须先确认PDF原文内容（日期、周数、学期），再进行后续处理。

### Step 2: 解析关键信息

从PDF中提取以下关键信息：

- **家书日期**：用于确定学期，从内容中搜索，**不要使用当前日期**
- **周数**：文件标题或内容包含了周数，例如：第xx周，注意**与封数区分**。
- **封数**：文件标题或内容包含了封数，例如：第xx封，注意**与周数区分**。
- **各科学系内容**：语文、数学、英语
- **周末作业**：独立作业和家长陪同作业

### Step 3: 计算学期和周数

根据家书日期计算目标目录

**学期计算规则**（2024年9月入学）：

- 2024年9月-2025年1月：1年级上
- 2025年2月-2025年7月：1年级下
- 2025年9月-2026年1月：2年级上
- 2026年2月-2026年7月：2年级下
- 以此类推，**年级用数字**...

### Step 4: 使用模板生成内容

严格按以下模板格式输出，**不添加任何额外内容**：

```markdown
---
date: YYYY年MM月DD日
---

## x年级上/下第xx周家书（第xx封）

### 学习内容
1. 按学科分列，每科详细列举本周所学知识点。
2. 语文列出课文名字，课文内容，其他所学知识。
3. 数学列出学习内容，知识点，方法论。
4. 英语列出单词，短语，句型，自然拼读内容。

#### 语文

- [从PDF提取的语文内容]

#### 数学

- [从PDF提取的数学内容]

#### 英语

- [从PDF提取的英语内容]

---

### 周末独立作业
1. 按**由易到难**排序。
2. **不包含**需家长陪同的作业（如亲子阅读，陪伴制作，辅助等）。
3. 英语、语文带有“复习”字样的作业备注"读3遍"（可独立完成）。
4. 英语带有”app“字样的作业备注”慢读，每个单词读标准“（可独立完成）。
5. 数学带有”复习“字样的作业备注"分析口算本和小练笔该部分的错题"（需家长陪同分析）。
6. 数学带有”计算练习“字样的作业备注”快速口算本做到所学单元“（可独立完成）。

| 顺序 | 学科 | 作业内容 |
| :-: | --- | --- |
| 1 | 学科 | 作业内容 |

### 家长陪同
1. **需要家长**陪同或配合的作业，逐个列出来，并详细说明需要家长配合什么。
2. 如有**考试或测验**（小达人就是考试），需要分析错题的，也在此列出。
3. **需要家长**准备的物品或事项，也在此列出。

- [从PDF提取的家长陪同内容]
```

### Step 5: 保存文件

将生成的内容保存到：

- 路径：`xinxin/school/{学期}/{第xx周}.md`
- 示例：`xinxin/school/2年级下/第14周.md`

## 重要提醒

1. **完全按照PDF内容**：总结时只使用PDF中实际存在的内容，不要添加推测或解释
2. **保持模板格式**：严格按照模板的结构和格式输出
3. **正确计算学期**：根据日期判断正确的学期
4. **周末独立作业**与**家长陪同**内容独立，严格区分作业或任务属于哪一种，不要两边同时列出。
