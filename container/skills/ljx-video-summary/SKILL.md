# 视频笔记技能

将抖音/小红书视频链接转换为结构化文字笔记。

## 工作流程

### 第一步：获取视频链接

用户会提供抖音或小红书的视频链接。

### 第二步：提取文字稿

调用 `mcp:wanyi-watermark` 工具的 `extract_douyin_text` 函数，传入视频链接获取文字稿。

返回结果是视频的逐字稿。

### 第三步：提炼生成笔记内容

根据提取的文字稿内容，进行以下工作：

- **视频内容大纲**：提炼主要内容，使用表格等工具让内容更直观
- **视频分析补充**：补充原作者的主要内容，可附带链接；相关知识科普；注意事项
- **相关内容推荐**：作者其他相关的作品；其他高质量相关内容推荐

### 第四步：按模板生成笔记

读取以下内容作为模板，将信息填充到模板中：

```
### 原视频标题

视频链接

作者名字

#### 视频内容大纲

- 主要内容提炼
- 可使用表格等工具让内容更直观
- 视频内容总结

#### 视频分析补充

- 原作者的主要内容补充，可附带链接
- 相关知识科普
- 注意事项

#### 原视频逐字稿

- 原始逐字稿

#### 相关内容推荐

- 作者其他相关的作品
- 其他高质量相关内容推荐
```

### 第五步：保存文件

- 文件保存到知识库，如果知识库不存在，先clone，如果存在，先pull。
- 文件名：提炼视频核心主题，控制在 5-10 个汉字，不得包含特殊字符（`/`、`\`、`:`、`*`、`?`、`"`、`<`、`>`、`|` 等）
- 保存路径：`pick/{提炼文件名}.md`
- 如果 pick 目录不存在，需先创建
- 创建完成后，提交到知识库。

## 注意事项

- 逐字稿部分直接使用提取的原始文字，不做修改
- 内容大纲和分析补充需要根据视频内容进行提炼总结
- 如果 MCP 调用失败，提示用户检查链接是否有效
- 文件名应简洁有意义，反映视频核心主题
